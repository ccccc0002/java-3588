# 二期协同执行协议（进度持久化 + 多 Agent 并行）

- 日期：2026-03-11
- 适用任务：`PHASE2-EXEC`
- 目标：保证阶段可见、节点可追踪、会话可恢复，避免重开任务丢失上下文。

## 1. 执行总则

1. 所有改动都要对应一个 `Phase`。
2. 每进入新阶段先更新 `phase2-status.json` 为 `in_progress`。
3. 每个重要节点必须保存 checkpoint。
4. 每个阶段结束必须做 compact，生成压缩上下文。
5. 每个阶段必须有“可见可验证结果”，不允许长期只改底层无输出。

## 2. 进度状态文件

1. 机器可读状态：[`phase2-status.json`](/D:/yihecode-server-master/yihecode-server-master/state/progress/phase2-status.json)
2. 人类可读看板：[`phase2-status.md`](/D:/yihecode-server-master/yihecode-server-master/state/progress/phase2-status.md)

状态取值：
1. `pending`
2. `in_progress`
3. `completed`
4. `blocked`

## 3. 必须执行的节点保存命令

### 3.1 阶段开始

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\collab\Update-PhaseStatus.ps1 `
  -TaskId PHASE2-EXEC -PhaseId PhaseX -Status in_progress -Summary "phase start"
```

### 3.2 重要节点保存

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\collab\Save-ImportantNode.ps1 `
  -TaskId PHASE2-EXEC -PhaseId PhaseX -Summary "milestone summary"
```

### 3.3 人工 compact（会话压缩）

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\collab\Compact-Session.ps1 `
  -TaskId PHASE2-EXEC -Stage PhaseX -Summary "manual compact"
```

### 3.4 阶段完成

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\collab\Update-PhaseStatus.ps1 `
  -TaskId PHASE2-EXEC -PhaseId PhaseX -Status completed -Summary "phase done"
```

## 4. 会话压缩策略（compact）

1. 每个阶段至少 compact 1 次（阶段结束时强制）。
2. 每完成 2-3 个关键节点再 compact 1 次。
3. 遇到阻塞（blocked）时立即 compact。
4. 每日收工前强制 compact 1 次。

compact 产物位置：
1. [`compact-latest.md`](/D:/yihecode-server-master/yihecode-server-master/state/compact/compact-latest.md)

## 5. 多 Agent 并行协作规则

1. `architect-agent`: 架构与边界。
2. `planner-agent`: 阶段推进与依赖调度。
3. `frontend-agent`: 首页驾驶舱 + 后台前端。
4. `backend-agent`: 接口、服务、数据模型。
5. `ai-runtime-agent`: 插件、推理、抽帧策略。
6. `media-agent`: ZLM、MPP+RGA、H264/H265。
7. `integration-agent`: 联调与端到端验证。
8. `qa-agent`: 测试与回归。
9. `ops-agent`: 部署、脚本、运行目录。
10. `security-and-audit-agent`: RBAC、日志、授权校验。

并行约束：
1. 不同 agent 优先操作不同文件域。
2. 任一高风险改动合并前必须有 checkpoint。
3. 每次冲突处理后必须再次保存 checkpoint。

## 8. 每阶段输出最低要求

每个阶段至少输出以下 4 项：
1. 功能结果说明。
2. 验证证据（测试/截图/接口返回/日志）。
3. 资源健康说明（CPU/内存关键观察）。
4. checkpoint 与 compact 记录。

## 9. 阻塞处理

当阶段阻塞时：
1. 更新 phase 状态为 `blocked`。
2. 保存重要节点（含阻塞原因）。
3. 执行 compact。
4. 在解除阻塞后更新状态并恢复执行。
